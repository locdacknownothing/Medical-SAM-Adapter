# Asset Locations Guide

This document describes where different assets (checkpoints, training logs, validation samples, and TensorBoard events) are saved after running training or validation scripts in this codebase.

---

## 1. Experiment Logs, Checkpoints, and Visualizations (`logs/`)

Both `train.py` and `val.py` initialize directories under `logs/` using the `set_log_dir` function defined in [utils.py](../utils.py).

Whenever you run a training or validation task with an experiment name (e.g., `-exp_name msa_test_stare`), a root experiment directory is created at:

```
logs/{args.exp_name}_{YYYY_MM_DD_HH_MM_SS}/
```

Inside this experiment directory, the following subfolders are created:

### 📁 `Model/`

- **Path**: `logs/{args.exp_name}_{timestamp}/Model/`
- **Contents**: Saved model weights/checkpoints.
  - `best_dice_checkpoint.pth` (saved whenever the model achieves a better validation Dice score).
  - `checkpoint_best.pth` (a copy of the best model checkpoint).

### 📁 `Log/`

- **Path**: `logs/{args.exp_name}_{timestamp}/Log/`
- **Contents**: Text logs generated by the Python logger detailing arguments, epoch-by-epoch training loss, validation Dice score, and IOU.

### 📁 `Samples/`

- **Path**: `logs/{args.exp_name}_{timestamp}/Samples/`
- **Contents**: Visualization images generated during validation rounds.
  - Naming pattern: `Test{image_names}+epoch+{epoch}.jpg`

---

## 2. TensorBoard Logs (`runs/`)

During training, TensorBoard event files are written using the `SummaryWriter` initialized in [train.py](../train.py).

- **Path**: `runs/{args.net}/{settings.TIME_NOW}/`
  - `{args.net}` defaults to `sam`.
  - `{settings.TIME_NOW}` is formatted as `YYYY-MM-DD_HH-MM-SS.ffffff` (microseconds resolution), defined in [conf/global_settings.py](../conf/global_settings.py).
- **Contents**: TensorBoard event files (`events.out.tfevents.*`) representing metrics like loss, dice score, and IOU over training steps/epochs.
